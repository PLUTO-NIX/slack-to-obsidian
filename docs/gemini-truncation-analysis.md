# Gemini 요약 과도 절단 원인 분석

## 현상
- **입력**: "리니어에서 triage 이슈 정리하고, 글로벌 프로젝트 이슈 점검"
- **출력**: "리니어에서 triage 이슈" (뒷부분 누락)

## 가능한 원인

### 1. parts[0]만 사용 (가능성 높음)
Gemini 2.5 Flash는 **thinking** 모델. 응답 구조:
- `parts[0]`: thought summary (추론 요약, 불완전할 수 있음)
- `parts[1]`: 실제 최종 답변

현재 코드가 `parts[0]`만 읽어 **thought 부분**을 가져오고 있을 가능성.

### 2. "요약" 프롬프트 의미
LLM이 "요약"을 "짧게 줄인다"로 해석 → 앞부분만 남기고 잘라냄.

### 3. Thinking 모드
Thinking이 활성화되면 응답이 여러 part로 분리. 단순 텍스트 생성에는 비활성화가 유리.

## 수정 방향
1. **모든 parts 텍스트 병합** 또는 **thought가 아닌 part만 추출**
2. **프롬프트**: "요약" → "다듬어/재작성", "생략 금지" 강조
3. **thinkingConfig**: thinking 비활성화 (지원 시)
